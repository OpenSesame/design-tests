# 11 — Phased Product Roadmap

---

## Roadmap Philosophy

The roadmap is organized around three horizons:

| Phase | Name | Goal |
|---|---|---|
| **Crawl** | MVP | Establish core product truth — prove that motivation-first design creates meaningfully better engagement |
| **Walk** | Growth | Expand the intelligence and stickiness layer — ambient signals, executive layer, deeper ROI data |
| **Run** | Enterprise | Achieve full vision — AI-generated content, industry benchmarking, complete enterprise integration ecosystem |

**Each phase must validate its core assumptions before the next begins.** The roadmap is not a fixed timeline — it is a capability progression. Phase 2 doesn't start because 6 months have passed; it starts because Phase 1's hypotheses have been validated.

## Phase 1 — Crawl (MVP)
### *Establish Core Product Truth*
**Target:** Months 1–6

**Hypothesis to validate:** Curation + personalization + motivation-first design creates meaningfully better learner engagement than existing solutions. The HCA framework resonates with executive buyers.

### Learner Experience
- [ ] The Feed — unified required + recommended content with algorithm personalization
- [ ] Career Identity language layer — motivation-first framing in all system communications
- [ ] Basic social proof signals — peer activity indicators in the feed
- [ ] Progressive onboarding — self-discovery tone, minimum viable data collection, rapid first-value delivery

### Manager Surface
- [ ] Manager Dashboard v1 — team completion visibility and basic assignment tools
- [ ] Manager's personal learning feed — leadership content alongside team tools

### Admin Surface
- [ ] Admin Console v1 — compliance tracking, program management, exportable reporting
- [ ] Bulk assignment and deadline management
- [ ] Basic re-engagement controls (contextual, not generic)

### Buyer Surface
- [ ] HCA Dashboard v1 — leading indicators, capability velocity, basic organizational health
- [ ] Dual indicator layer — leading vs. lagging clearly labeled

### Platform & Infrastructure
- [ ] Privacy architecture — tiered visibility model fully implemented from day one
- [ ] Progressive profiling system — behavior-based data enrichment over time
- [ ] AI partner integration — recommendation and personalization layer
- [ ] Integrations: BambooHR, Slack, Google Workspace

### Benchmark Infrastructure (invisible to users)
- [ ] Anonymized aggregate data pipeline — collecting from day one for future benchmark dataset
- [ ] Data schema designed to support future industry benchmarking

## Feature Prioritization Principles

When new feature requests arise, evaluate them against these questions in order:

1. **Does it serve the Learner's motivation or relevance?** If yes, high priority.
2. **Does it serve the Buyer's ROI story?** If yes, high priority.
3. **Does it dilute the product identity?** If yes, deprioritize regardless of appeal.
4. **Can it be scoped as a feed content type rather than a new surface?** If yes, prefer that path.
5. **Does it require infrastructure that isn't built yet?** If yes, roadmap it correctly rather than building prematurely.

### The Scope Creep Watchlist
These features are interesting but must be carefully scoped to avoid diluting the product:

- **Career recommendation / job matching** — Career Identity Engine should inform learning, not replace a career platform
- **Performance management** — ILS informs performance data, it does not own performance reviews
- **Full social networking** — Social proof signals are not building a LinkedIn competitor
- **Talent marketplace** — Adjacent to Phase 3 territory but not a Phase 1–2 consideration
